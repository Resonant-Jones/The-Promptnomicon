# Project Reality Footer

Use this footer at the end of changelogs, dev logs, receipts, roadmap updates, and agent sessions.

The goal is to leave durable breadcrumbs that can later become a project graph.

```text
PROJECT REALITY STATE

Date:
Repo:
Branch:
Commit:
Author/operator:
Session type:

Changed files:
- 

What changed:
- 

Why it changed:
- 

Implemented:
- 

Planned only:
- 

Validated:
- 

Runtime proof:
- 

Known unstable areas:
- 

Risks introduced:
- 

Docs updated:
- 

Docs possibly drifting:
- 

Next safe task:
- 

Open questions:
- 

Risk level: low | medium | high | unknown
```

## Guidance

Keep it lightweight. This is not an essay.

A useful footer is better than a perfect footer. The purpose is to make future coordination easier, not to punish anyone for forgetting details.

## Future Graph Fields

These fields can later be parsed into a graph:

- repo
- branch
- commit
- files changed
- feature area
- decision references
- validation references
- risk labels
- next task
- unresolved questions

The footer is the seed. The graph can grow later.
